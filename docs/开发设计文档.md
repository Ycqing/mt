# 抖音商家数据分析系统 - 开发设计文档

## 1. 项目概述

### 1.1 项目背景
自用抖音商家数据分析系统，用于收集、整理、分析商家多平台数据，输出营销报告。

### 1.2 核心目标
- 集中管理商家基础信息（名称、官网、社交账号、上市状态等）
- 后台自动采集公开数据（官网信息、企业工商数据、产品评价等）
- 接入第三方数据平台（飞瓜、蝉妈妈、发现报告）
- 支持商家横向对比分析
- 自动生成标准化营销报告

### 1.3 技术选型

| 层级 | 技术选型 | 说明 |
|------|----------|------|
| 前端 | **Vue 3 + Vite + Element Plus** | 现代化响应式 UI，组件丰富 |
| 后端 | **Spring Boot 3 + MyBatis-Plus** | 稳定生态，ORM 强大 |
| 数据库 | **MySQL 8.0** | 关系型数据存储，支持复杂查询 |
| 爬虫 | **Spring Boot + Jsoup + HttpClient** | 基础页面数据采集 |
| 报告 | **Apache POI + iText** | Excel/PDF 报告生成 |
| 定时任务 | **XXL-Job** | 分布式任务调度 |

| POST | `/api/merchant/{id}/operating-data` | 手动录入经营数据 | OperatingDataDTO |
| GET | `/api/merchant/{id}/financial-data` | 获取财务数据 | merchantId, year |
| POST | `/api/merchant/{id}/financial-data` | 手动录入财务数据 | FinancialDataDTO |

### 5.3 定时任务设计

| 任务名称 | Cron表达式 | 说明 |
|----------|-----------|------|
| socialDataSync | `0 0 2 * * ?` | 每天凌晨2点同步社交账号数据 |

## 6. 爬虫策略设计

### 6.1 采集流程

```
用户发起采集请求
      │
      ▼
创建 FetchTask 记录 (状态: pending)
      │
      ▼
任务调度器领取任务 (状态: running)
      │
      ▼
┌─────┴─────┐
│ 采集源判断 │
└─────┬─────┘
      │
      ├──→ 官网采集 ──→ Jsoup 解析 HTML ──→ 提取结构化数据
      │
      ├──→ 财报采集 ──→ 巨潮资讯/东方财富 API ──→ 解析 PDF/HTML
      │
      ├──→ 评价采集 ──→ 平台搜索 API / 爬虫 ──→ 情感分析处理
      │
      ├──→ 第三方API ──→ 飞瓜/蝉妈妈 API ──→ 数据格式化入库
      │
      └──→ 社交账号 ──→ 平台公开数据 ──→ 粉丝数/帖子数

      │
      ▼
数据入库 (merchant / social_accounts / product / review ...)
      │
      ▼
更新 FetchTask (状态: success, result)
      │
      ▼
推送通知 (WebSocket / 邮件) [可选]
```

### 6.2 反爬应对策略

| 策略 | 实现方式 |
|------|----------|
| 请求限速 | 单IP每分钟请求不超过60次 |
| UA轮换 | 维护User-Agent池，每次随机选取 |
| 代理IP池 | 使用代理IP轮换访问 (高成本，可选) |
| 请求间隔 | 相邻请求间隔3-10秒随机延时 |
| Cookie管理 | 模拟登录态，Cookie定期刷新 |
| 验证码处理 | 接入打码平台 (可选) |
| 失败重试 | HTTP 403/429/500 时自动重试，最多3次 |
